Excel Template Tips

How to fill out the template correctly

Guides › Excel Template Tips

Template Overview

The Excel template is designed to let you easily enter multiple sequences at once. Each row represents one sequence, and the columns cover all fields required by ST.26.

Required Fields

  • SEQ ID NO: Sequential number starting from 1 (auto-assigned if left blank)
  • Molecule Type: DNA, RNA, or AA (amino acid)
  • Organism Name: Scientific name of the source organism (e.g., Homo sapiens, Mus musculus)
  • Sequence: The actual nucleotide or amino acid sequence
Tip: Refer to the example rows included in the template. They provide correctly formatted entries for DNA, RNA, and protein sequences.

Sequence Format Rules

DNA / RNA Sequences

  • Use IUPAC single-letter codes: a, t, g, c
  • RNA sequences also use t instead of u (applies to both WIPO Sequence and this template)
  • Both uppercase and lowercase are accepted (auto-converted)
  • Spaces, line breaks, and numbers are automatically removed
  • A minimum of 10 nucleotides is required per WIPO ST.26 standard
Important: Even for RNA sequences, enter t instead of u. WIPO Sequence uses thymine notation for both DNA and RNA in sequence listings.

Amino Acid Sequences

  • Use single-letter amino acid codes (e.g., M, A, K, L)
  • A minimum of 4 amino acids is required per WIPO ST.26 standard
  • Use X for unknown amino acids
Common Mistake: Do not include header characters like ">" (FASTA format headers) in the sequence field. Only pure sequence characters should be entered.

Organism Name

Organism names must follow NCBI taxonomy naming conventions:

  • Use the full scientific name (e.g., Homo sapiens instead of "Human")
  • Use Artificial Sequence for synthetic/artificial sequences
  • Use Unknown for unidentified organisms
  • Case is significant and will be validated against NCBI taxonomy

Feature Annotations (Optional)

You can add feature annotations to describe functional regions of a sequence:

  • Feature Key: e.g., CDS, misc_feature, source
  • Location: Position range (e.g., 1..300)
  • Description: A brief description of the feature

Tips for Large Datasets

  • The converter can handle 1,000+ sequences in a single file
  • For very large files, conversion may take up to 1–2 minutes
  • For best performance, keep Excel file size under 10 MB
  • If you have more than 5,000 sequences, consider splitting them into multiple files
Tip: After conversion, always validate the XML in WIPO Sequence before filing with the patent office. The validator will catch any remaining issues.
