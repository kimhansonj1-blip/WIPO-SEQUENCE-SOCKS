Conseils pour le modele Excel

Comment remplir le modele correctement

Guides › Conseils pour le modele Excel

Apercu du modele

Notre modele Excel est concu pour faciliter la saisie de plusieurs sequences a la fois. Chaque ligne represente une sequence et les colonnes capturent tous les champs requis par ST.26.

Champs obligatoires

  • SEQ ID NO : Numero sequentiel commencant a 1 (attribue automatiquement si laisse vide)
  • Type de molecule : DNA, RNA ou AA (acide amine)
  • Nom de l'organisme : Nom scientifique de l'organisme source (ex. : Homo sapiens, Mus musculus)
  • Sequence : La sequence nucleotidique ou en acides amines proprement dite
Conseil : Consultez les lignes d'exemple incluses dans le modele. Elles montrent des entrees correctement formatees pour des sequences d'ADN, d'ARN et de proteines.

Regles de format des sequences

Sequences ADN / ARN

  • Utilisez les codes IUPAC a une seule lettre : a, t, g, c
  • Les sequences ARN utilisent egalement t a la place de u (dans WIPO Sequence et dans notre modele)
  • Minuscules ou majuscules fonctionnent indifferemment (converties automatiquement)
  • Les espaces, sauts de ligne et chiffres sont automatiquement supprimes
  • Minimum 10 nucleotides requis par la norme WIPO ST.26
Important : Meme pour les sequences ARN, ecrivez t et non u. WIPO Sequence utilise la notation thymine pour l'ADN et l'ARN dans le listage de sequences.

Sequences d'acides amines

  • Utilisez les codes a une lettre pour les acides amines (ex. : M, A, K, L)
  • Minimum 4 acides amines requis par la norme WIPO ST.26
  • Utilisez X pour les acides amines inconnus ou non specifies
Erreur courante : N'incluez pas de caracteres d'en-tete comme ">" (en-tetes format FASTA) dans le champ sequence. Seuls les caracteres bruts de la sequence doivent etre saisis.

Nom de l'organisme

Le nom de l'organisme doit suivre les conventions de nomenclature taxonomique du NCBI :

  • Utilisez le nom scientifique complet (ex. : Homo sapiens, pas "Humain")
  • Pour les sequences artificielles ou synthetiques, utilisez Artificial Sequence
  • Pour les organismes inconnus, utilisez Unknown
  • Le nom est sensible a la casse tel qu'affiche, mais valide par rapport a la taxonomie NCBI

Annotations de caracteristiques (optionnel)

Vous pouvez ajouter des annotations de caracteristiques pour decrire les regions fonctionnelles de vos sequences :

  • Cle de caracteristique : ex. : CDS, misc_feature, source
  • Localisation : Plage de position (ex. : 1..300)
  • Description : Breve description de la caracteristique

Conseils pour les grands ensembles de donnees

  • Notre convertisseur traite plus de 1 000 sequences dans un seul fichier
  • Pour les fichiers tres volumineux, la conversion peut prendre jusqu'a 1-2 minutes
  • Maintenez le fichier Excel sous 10 Mo pour des performances optimales
  • Si vous avez plus de 5 000 sequences, envisagez de les diviser en plusieurs fichiers
Conseil : Apres la conversion, validez toujours votre XML dans WIPO Sequence avant de le soumettre a un office de brevets. Le validateur detectera les problemes restants.
