템플릿 개요

본 엑셀 템플릿은 여러 서열을 한 번에 쉽게 입력할 수 있도록 설계되었습니다. 각 행은 하나의 서열을 나타내며, 열은 ST.26에서 요구하는 모든 필드를 포함합니다.

필수 입력 항목

  • SEQ ID NO: 1부터 시작하는 순차 번호 (비워두면 자동 부여)
  • Molecule Type: DNA, RNA, 또는 AA (아미노산)
  • Organism Name: 출처 생물체의 학명 (예: Homo sapiens, Mus musculus)
  • Sequence: 실제 뉴클레오타이드 또는 아미노산 서열
팁: 템플릿에 포함된 예시 행을 참고하세요. DNA, RNA, 단백질 서열에 대한 올바른 형식의 입력 예시가 제공됩니다.

서열 형식 규칙

DNA / RNA 서열

  • IUPAC 단일 문자 코드를 사용합니다: a, t, g, c
  • RNA 서열도 u 대신 t를 사용합니다 (WIPO Sequence 및 본 템플릿 모두 동일)
  • 대문자와 소문자 모두 허용됩니다 (자동 변환됨)
  • 공백, 줄바꿈, 숫자는 자동으로 제거됩니다
  • WIPO ST.26 표준에 따라 최소 10개의 뉴클레오타이드가 필요합니다
중요: RNA 서열이더라도 u가 아닌 t를 입력해야 합니다. WIPO Sequence는 서열목록에서 DNA와 RNA 모두 티민 표기법을 사용합니다.

아미노산 서열

  • 단일 문자 아미노산 코드를 사용합니다 (예: M, A, K, L)
  • WIPO ST.26 표준에 따라 최소 4개의 아미노산이 필요합니다
  • 미지정 아미노산에는 X를 사용합니다
자주 발생하는 실수: 서열 필드에 ">" 같은 헤더 문자(FASTA 형식 헤더)를 포함하지 마세요. 순수 서열 문자만 입력해야 합니다.

생물체 학명 (Organism Name)

생물체 학명은 NCBI 분류 명명 규칙을 따라야 합니다:

  • 전체 학명을 사용합니다 (예: "Human" 대신 Homo sapiens)
  • 인공/합성 서열의 경우 Artificial Sequence를 사용합니다
  • 미지 생물체의 경우 Unknown을 사용합니다
  • 표시 시 대소문자를 구분하며 NCBI 분류에 따라 검증됩니다

특징 주석 (선택사항)

서열의 기능적 영역을 설명하는 특징 주석을 추가할 수 있습니다:

  • Feature Key: 예: CDS, misc_feature, source
  • Location: 위치 범위 (예: 1..300)
  • Description: 특징에 대한 간단한 설명

대용량 데이터 팁

  • 본 변환기는 단일 파일에서 1,000개 이상의 서열을 처리할 수 있습니다
  • 매우 큰 파일의 경우 변환에 최대 1~2분이 소요될 수 있습니다
  • 최적의 성능을 위해 엑셀 파일 크기를 10MB 이하로 유지하세요
  • 서열이 5,000개를 초과하는 경우 여러 파일로 분할하는 것을 권장합니다
팁: 변환 후에는 특허청에 제출하기 전에 항상 WIPO Sequence에서 XML을 검증하세요. 검증기를 통해 나머지 문제를 확인할 수 있습니다.
